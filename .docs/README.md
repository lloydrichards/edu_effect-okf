## Purpose

These files are meant for re-reading and future context, not step-by-step teaching.
They summarize the stable ideas that emerged from the OKF exploration:

- what OKF is and is not
- how OKF can be projected into a graph
- how graph and RAG fit together
- what a useful CLI surface could look like

## Design Principles Captured Here

- OKF is a portable knowledge substrate
- vector retrieval finds entry points
- graph traversal expands structured context
- the CLI should expose bundle, concept, graph, query, and ingest workflows
